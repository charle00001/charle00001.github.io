在2023-10-22修正#

了解Hexo#

如果你也想做一个像这样的个人博客,请务必认真看完！！本文针对Mac,使用windows和Linux请自行上网查询！！

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
（好吧，Hexo官网是这么讲的）

它是个非常不错博客框架，有很多好看的主题可以应用在自己的博客上，这篇文章要讲述如何用hexo写博客并部署到github上就好了。

如果你已经用hexo写了博客并部署到github上了，就不要再看这篇文章了，快去 https://hexo.io/themes/ 寻找好看的、心仪的主题吧！！！

配置需要的环境#

先安装Node.js：#

https://nodejs.org/zh-cn/download/
下载好了后请自行根据提示安装，推荐LTS版本（长期支持版）。

安装git：#

Macos（好像有自带的，不过建议重新安装）:https://sourceforge.net/projects/git-osx-installer/

安装Hexo:#

官网给的是这条命令：
npm install -g hexo-cli

大家注意了，要使用管理员权限，在命令开头加上sudo空格。
npm install -g hexo-cli
回车，等待下载安装即可！
熟悉 npm 的进阶用户，可以仅局部安装 hexo 包。
npm install hexo

使用Hexo搭建博客#

在桌面新建一个空文件夹blog#

（因为我早就自己搭建过一个名字一样，所以使用blog2演示）

使用Hexo初始化博客#

首先进入blog文件夹：npx hexo init

启动hexo#

npx hexo s

如果提示4000端口被占用：

遇到这种情况的话，你肯定之前也运行过这个命令并且没有control+c
解决方法如下：
sudo lsof -i:4000，查看被占用进程的pid
再输sudo kill -9 pid （pid为上一步查看到的数字）
然后再继续npx hexo s，则成功。

在浏览器输入：http://localhost:4000，可查看到默认生成了一篇博文：

如何新建一篇博文？#

control+c断开刚刚的链接，新建一篇博文：
npx hexo n "发布第一篇个人博客"

更改博文内容：#

接下来就是更改博文内容，直接使用Visual Studio Code打开文章，找到博客根目录/source/_posts，里面有你创建的文章，直接做修改即可，不熟悉MarkDown格式的可以看看这个网站：
https://www.runoob.com/markdown/md-tutorial.html
（不难，现在我就在写这玩意）
随便使用哪个编辑器，反正我习惯用VSCode。

在本地预览：#

同样在blog2文件夹下：
清理public文件夹：npx hexo clean
生成：npx hexo g
启动：npx hexo s

刷新http://localhost:4000/则看到，内容已经更新上去了：

部署至github#

如果你想让别人也能访问到（废话），那么免费的方法就是把网站部署至github。
还有很多地方可以部署网站，这种云部署平台一搜一大把。我还推荐两个：Vercel和CF Pages。
这三个平台是绝对靠谱滴，微软、cloudflare。还有vercel，绝对是神器。他还可以部署【动态】网站，重点是【免费】！

登录github：#

https://github.com/
没账号的新建一个（这个我不用教了吧），新创建一个仓库：

仓库名必须为：昵称.github.io，输入描述（也可以不输），创建（记得选公开）：

（因为不能重名，而且我已经有了，所以有红色感叹号，你们不会）

安装git部署插件#

命令行下，ctrl+c，退出命令。还是在blog2目录下，安装git部署插件
npm install hexo-deployer-git --save

更改配置#

更改blog2文件夹根目录下，_config.yml的配置：

把这些替换为：

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:yourname/yourname.github.io.git
  branch: master

记得把yourname替换成你的昵称。

设置SSH密钥：#

首先检测自己电脑是否存在 SSH key：#

在生成SSH密钥之前，您可以检查是否有任何现有的SSH密钥。

打开终端#
查看是否存在现有SSH密钥#

输入命令以查看是否存在现有SSH密钥⬇️：

ls -al ~/.ssh
# 列出.ssh目录中的文件(如果存在的话)
检查目录列表以查看您是否已拥有公共SSH密钥。#

·如果你本地没有生成的话
终端上面会显示：No such file or directory

·如果已经存在的话
则会显示 id_rsa 和 id_rsa.pub

如果本地没有的话，我们要自己生成新的SSH秘钥（如果存在，请跳过4.4.1.）

创建一个新的ssh密钥#

在终端输入下面的命令，记得替换您的GitHub电子邮件地址,这将使用提供的电子邮件作为标签创建一个新的ssh密钥。

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
#"your_email@example.com"为示例邮箱，此处应填写为你在github上面注册的邮箱。

显示：

Generating public/private rsa key pair.
系统提示您“输入要保存密钥的文件”时#

当系统提示您“输入要保存密钥的文件”时，按Enter键使用默认文件位置。

Enter a file in which to save the key (/Users/you/.ssh/id_rsa): [Press enter]
键入安全密码#

在提示符下，键入安全密码（强烈不建议设置密码，否则你上传一次就要输一次，特别麻烦。。。）。可以连续Enter，不设置密码。

Enter passphrase (empty for no passphrase): [Type a passphrase]
Enter same passphrase again: [Type passphrase again]

成功后，终端显示：

Your identitification has been saved in /Users/xxx/.ssh/id_rsa.
Your public key has been saved in /user/xxx/id_rsa.pub.
the key fingerprint is xxxxxxxxxx…….xxxxxxxx your_email@example.com
The key’s randomart image is:

将SSH密钥添加到ssh-agent#

在后台启动ssh-agent#
输入命令：#

eval "$(ssh-agent -s)"

> Agent pid xxxxx
终端输入命令：#

ssh-add -K ~/.ssh/id_rsa

如果设置密码，此时会要求输入passphrase（输入步骤二中你当时设置的passphrase就好）
成功后，终端显示：

Identity added: /Users/xxx/.ssh/id_rsa (your_email@example.com)

添加SSH key 到Github 上#

将SSH密钥复制到剪贴板#

pbcopy < ~/.ssh/id_rsa.pub
复制id_rsa.pub的内容到剪贴板提示：如果pbcopy不起作用，您可以找到隐藏.ssh文件夹（路径~/.ssh/，在文本编辑器中打开id_rsa.pub文件，然后将文件内容复制到剪贴板。

找到“Settings”#

打开Github,在任意页面的右上角，单击您的个人资料照片，然后单击“Settings”。

找到“SSH and GPG keys”#

在用户设置侧栏中，单击“SSH and GPG keys”。

单击“ New SSH key”#

添加密钥#

在“Title”字段中添加描述性标签，Key字段中粘贴您复制的id_rsa.pub内容。最后点击“Add SSH key”

确认您的GitHub密码#

最后请确认您的GitHub密码。

如何检测SSH key#

终端输入命令#

ssh git@github.com

验证SSH key是否有权限访问你的github，输入“yes”

The authenticity of host ‘github.com (13.250.177.223)’ can’t be established.
RSA key fingerprint is SHA256:nThbg6kXUpJWGl7E1IGOCspRomTxdCARLviKw6E5SY8.
Are you sure you want to continue connecting (yes/no)? yes

成功会显示为

Hi you name! You’ve successfully authenticated, but GitHub does not provide shell access.

push至GitHub仓库#

在部署到github之前，别忘了要先#

npx hexo clean
# 删除之前编译的文件（public、database等）
npx hexo g
# 重新编译新文件等待被部署至github
# 部署完了之后就可以运行npx hexo clean把编译的文件删掉了

现在，就可以push至GitHub远程仓库了#

npx hexo d
#d就是deploy，即部署
